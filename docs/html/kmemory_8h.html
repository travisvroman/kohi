Detailed Description

This file contains the structures and functions of the memory system. This is responsible for memory interaction with the platform layer, such as allocations/frees and tagging of memory allocations.

Author
Travis Vroman (travi.nosp@m.s@ko.nosp@m.hieng.nosp@m.ine..nosp@m.com)
Note
Note that reliance on this will likely be by core systems only, as items using allocations directly will use allocators as they are added to the system.

1.0

KAPI void memory_system_initialize ( u64 *  memory_requirement,
void *  state 
)

Initializes the memory system. Should be called twice, once to obtain the memory requirement (passing state=0), and a second time with state recieving an allocated block of memory.

Parameters
memory_requirementA pointer to hold the memory requirement in bytes, used for allocation.
stateA block of memory to hold the state for this system. Can be 0 when just obtaining requirement.
